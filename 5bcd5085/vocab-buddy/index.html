Vocab Buddy

绪论

研究背景

随着互联网的普及，网络教育已经成为一种越来越流行的教育方式。在线词汇学习已经成为许多学习者提高英语能力的首选方式。在线生词默写系统是一种帮助学生在不同语境下学习和巩固单词记忆的有效方式。

近年来，随着互联网技术的发展和网络教育的不断普及，越来越多的学生选择使用在线教育资源来提高自己的英语能力。而在线生词默写系统是一种基于 Web 的在线学习工具，它可以帮助学生提高词汇学习效果。这种系统通常由两部分组成：学习和测试。学习部分通过展示单词的拼写、发音、词义和用法等信息，帮助学生了解单词的基本信息。测试部分则通过要求学生听写、拼写、填空、选择等方式来检测学生对单词的掌握程度。这种方式可以帮助学生在不同语境下学习和巩固单词记忆，提高英语水平。

在线生词默写系统采用了一种互动式的学习方式，可以帮助学生更好地理解和掌握单词。这种学习方式不仅可以提高学生的学习兴趣，还可以激发他们学习的动力。通过在线学习，学生可以随时随地进行学习，节约了时间和成本。通过在线测试，学生也可以及时了解自己的学习情况并纠正错误，提高学习效果。

同时，在线生词默写系统还可以方便教师管理学生的学习情况。教师可以通过该系统跟踪学生的学习进度和学习成果，了解学生掌握单词的情况，及时调整教学内容和方法，提高教学效果。

在线生词默写系统是一种创新的学习方式，可以为教育教学改革提供支持。随着信息技术的发展，传统的教学方式已经无法满足学生的需求。在线学习系统可以通过多种方式展示单词的信息，使学生能够在不同的环境下进行学习，帮助学生更好地理解和掌握单词。这种学习方式也符合现代学生的学习习惯和需求，有助于促进教育教学改革的发展。

随着全球化的发展，英语已经成为一种国际通用语言。因此，提高英语教育水平已经成为国家教育发展的重要任务。在线生词默写系统作为一种新的学习方式，可以帮助学生更好地掌握英语单词，提高英语水平，进而提高国家的英语教育水平。

基于 Web 的在线生词默写系统的优势和意义不仅仅局限于学生和教师个体层面，同时也在社会层面有着重要的意义。随着全球化的不断推进，英语已经成为全球通用的语言之一，掌握英语已经成为个人和国家竞争力的体现。而在线生词默写系统提供了一种先进的学习方式，通过在不同环境下展示单词的信息，帮助学生更好地掌握和应用英语单词，进而提高英语水平。提高英语水平不仅可以提高个人的职业竞争力，还可以促进国际交流和合作。因此，研究和应用基于 Web 的在线生词默写系统具有重要的现实意义和发展前景，有助于推动英语教育的发展和提高国家的竞争力。

国内外研究现状

在2005年，学者高李冬和魏育红提出将语音、语境与生词本相结合的方法来背单词。通过将单词联系在一起，以增量记忆和比较辨异的方式来有效背诵单词。

2019年，学者蒋惠华进行了研究，发现学生们虽然更喜欢教师带领式的单词听写，但使用文字转语音（Text-to-Speech, TTS）技术后，成绩却更高。这项研究在电子计算机辅助语言学习方面取得了新的进展。她认为学生对新技术并不感冒是因为传统教学方式以教师为中心。因此，她建议在语言学习课堂上适当使用 TTS 技术进行教学，以提高教师自身的学习能力。她的研究为这一领域带来了新的突破，解决了学生不太欢迎新技术到来的问题。

国外学者 Zuraina Ali、Norsuhaily Abu Bakar、Waqar Ahmad 和 Jumadil Saputra 在2022年提出了网络游戏对学生记单词有帮助的想法，并进行了深入研究。在研究中，他们发现使用定量研究设计，可以提高学生在短期和长期词汇保留方面的表现。这项研究为教学方式提供了启示，在通过 IT 技术帮助学生记单词的方面取得了新的突破，使得该领域的研究有了新的进展。研究中提出了解决电子计算机技术在传统语言课堂上不受欢迎的问题的解决方案。

主要设计内容

本设计将采用前后端分离的架构模式，对在线生词默写系统进行分析与设计，使用 Java 语言搭配 JetBrains IntelliJ IDEA 工具和 TypeScript 语言与 JetBrains WebStorm 工具相结合的方式编写系统实现代码，后端系统使用 Spring Boot 3 作为主体框架、前端系统采用渐进式 Vue.js 框架进行实现。具体设计内容如下：

  1. 对听写测试的生成策略进行深入学习与研究。参照德国著名心理学家艾宾浩斯对记忆遗忘规律的研究，如果想将单词进行长期记忆，就需要通过在不断的训练之间加上一顶的间隔；
  2. 对在线生词默写系统中需要重要技术进行学习与研究。搭配 MyBatis Plus 的 Spring Boot 框架的本质是 MVC
    框架，通过使用该框架可以有效降低整个系统中模型层（Model）、视图层（View）和控制器层（Controller）的耦合度，提高系统的扩展性与可维护性；前端采用数据驱动的轻量渐进式
    Vue.js 框架，搭配对于数据进行约束的 TypeScript 语言进行开发，并使用 Axios 对接前后端数据。后端将采用全新的 Gradle
    工具进行项目依赖与生命周期的管理，前端采用 pnpm 工具进行项目依赖与生命周期的管理，并使用 Git
    代码工具进行代码版本维护与管理，并将源代码托管在腾讯云 Coding 协作平台；
  3. 对在线生词默写系统进行需求分析，对系统所具备的核心功能进行分析总结，并对系统的性能以及系统应该具备的安全性进行分析与总结，同时借助
    XMind 与 Microsoft Visio 绘制相关图表进行辅助说明；
  4. 对在线生词默写系统进行总体设计，主要包括对系统的架构和功能结构进行设计，并针对不同的功能模块进行详细设计、对系统进行数据库设计，设计出系统所需要使用的实体结构，并整理不同实体之间的关系，借助
    Navicat Data Modeler 工具绘制出其逻辑模型并进行物理数据库管理、对系统进行安全性设计，采用分层结构、多重验证等多种措施保证系统安全；
  5. 实现在线生词默写系统，借助智能集成开发环境以及系统设计描述对系统的核心功能及安全性进行编码，并通过对各个模块关键功能的业务逻辑实现描述的功能，借助绘图工具绘制的相关图表来说明整个系统功能实现以及安全实现的过程；
  6. 对系统进行测试，通过黑盒测试和白盒测试对系统进行功能和代码结构测试，通过压力测试与渗透测试测试系统的性能及安全性，根据最终测试结果讨论系统的后续维护。

通过设计具有互动性、自适应性和个性化等特点的计算机程序，提高学生的学习兴趣和主动性，提升他们的学习效果和成绩。计算机程序可以为学生提供更多的练习机会，并帮助学生更好地理解和掌握单词的意思和用法。通过本文的研究和设计，希望为教育教学中如何更好地利用计算机提供一种新的思路和方案。

系统分析与设计

系统需求分析

系统需求概述

该系统主要是为学生，尤其是小学生，提供一个在线生词默写平台，通过融合单词记忆和有趣的类似游戏的挑战模式，辅助用户记单词。

本系统应该有用户注册、用户登录、用户注销等基本功能，同时为用户提供生成听写以及在线进行听写测试的功能。生成听写测试功能将根据用户的选择进行客制化生成，用户可以自定义选择生成模式：如基于某一单词标签随机抽取指定数量的单词生成、基于以往听写的记录生成等。对在线进行默写/听写测试功能，考虑到该系统的主要目标用户为学生，并且不同学生对于计算机的操作熟练度不同，因此本系统需要通过用户注册功能收集用户的部分关键信息，如年级等，并根据用户的这些关键信息或用户自己的选择来决定用户答题的方式，如选择题、填空题或完整拼写等不同答题方式。针对管理员用户，其可以对词库进行管理，包括但不限于上传词库、创建标签、标记单词等功能。

除以上针对系统功能性需求之外，系统还应该保证运行的稳定性，确保用户在使用过程中不会出现或者尽量少地出现系统崩溃地现象；同时系统应该兼具易用性，要符合操作地直觉，确保绝大部分用户都能够在没有操作说明书地情况下使用系统地核心功能；此外，由于本系统需要收集部分用户地隐私信息，因此需要确保本系统地安全性；由于系统设计可能不够成熟，因此要保证后期地可维护性与可扩展性，保证可以扩展出更多需要的功能。

系统功能需求

从上述对于需求的描述中，我们可以推断出该系统的主要功能有用户信息管理、创建单词默写测试、参加在线单词默写测试、保存历史默写测试以及词库管理等功能。我们从系统的大致是用描述来说明这些功能模块：

  1. 用户信息管理：进入系统主页面，用户在注册登录操作之后，用户可以通过该功能查看和修改其个人信息；
  2. 创建单词默写测试：进入系统主页面，用户在注册登录操作之后，其可以根据自己的需要选择词库并根据自己的需要选择创建一个听写测试。
  3. 参加单词默写测试：在创建听写测试后，用户可以选择打印该听写测试或线上参加听写测试。若用户参加了线上听写测试，将会自动保存单词默写或听写的情况。
  4. 保存历史默写测试：若用户选择参加线上用户测试，那么在用户参与测试时便会自动保存所有默写或听写的详情；如果用户选择了打印测试，那么该功能需要在用户上传默写或听写测试卷，系统根据用户的测试结果进行分析。
  5. 分享默写或听写测试：用户可以选择生成一张相关图片，其中包含有被分享听写测试的详细信息，例如被测试者的昵称或姓名、被测试的单词、用户的拼写等等，让学生分享自己成功完成单词听写默写测试的喜悦，并可以促进其同学们的竞争意识，强化同学圈子内的整体英语单词词汇量。
  6. 词库管理功能：系统管理员登录系统后，可以选择上传包含有单词内容的数据文件，系统将根据情况自动分析该文件的单词并将其保存到系统内部，允许其他用户进行访问，保证系统内词库的及时性。

系统用户及用例分析

从上述需求描述中，分析出该系统内主要用户角色有管理员及用户。其中用户主要包含使用该系统的人员，其主体为学生，并有概率存在一部分家长。系统管理员则主要是系统的管理人员。由于本系统并不针对某一学校进行开发，因此系统管理员会选择可信任的相关系统运维人员以及通过审核的一些老师。针对管理员角色，也被分成拥有不同权限的管理员：一种拥有系统最高控制权限的系统管理员，一种专门负责单词信息维护的信息维护员。虽然两者都是管理员，但是后者仅可以进行例如词库维护的内容。

管理员

系统管理员对大部分功能都具有操作权限，但是本质上只是进行信息审计有关的操作。对于教师管理员，主要是提供其进行词库维护的权限。据此分析，管理员的主要活动用例图如图所示管理员的活动用例说明：

  1. 词库的维护，主要是对于标签的管理以及单词、单词释义的管理；
  2. 用户审计，对注册的用户进行审查；封禁违规使用的账户；帮助用户维护账户。

普通用户

用户是本系统的主要操作角色。其在使用系统时，有其自己账户所生成听写的操作权限，例如：创建单词默写测试、参加单词默写测试、保存历史默写测试、分享默写或听写测试以及个人信息维护等。据此分析，管理员的主要活动用例图如图所示。

用户的主要活动用例说明：

  1. 创建单词默写测试，用户可以根据自己的需要创建听写或默写测试，可以通过自定义参数设置听写的生成模式；
  2. 参加单词默写测试，用户可以参加自己创建的听写测试，通过在线的方式参加该测试，测试自己的听写水平；
  3. 保存历史默写测试，用户通过在线参加听写默写测试，系统将会自动保存用户的听写历史记录，以供用户后续进行复习使用；
  4. 分享默写或听写测试，用户可以分享其参加过的听写或默写测试，其他人可以直接访问其参加的听写，并看到该用户做出的答卷；
  5. 个人信息维护，用户可以管理、修改自己的个人信息。

系统性能需求分析

本系统主要面向学校师生，从执行质量的角度来看，要求系统具备良好的性能、易用性和稳定性；从发展质量的角度来看，要求系统具备可扩展性和可维护性。具体的系统性能需求要求如下表所示。

特性名称 需求说明
性能 针对响应时间的要求，遵循二五十原则可作为普遍的标准。
当用户在2秒以内得到响应时，系统的响应速度被认为是很快的；
当用户在2-5秒内得到响应时，系统的响应速度在可接受的范围内；
当用户在5-10秒内得到响应时，系统的响应速度被视为较慢，可能会引起用户的不满和不良体验；
超过10秒后仍然无法得到响应时，系统的响应速度被认为是严重滞后的，这将严重影响用户的体验和满意度。特别地，在并发操作用户量达到8000的情况下，操作的响应时间应至少小于10秒。
易用性 系统界面设计应当具备以下特点：功能结构清晰，导航设置合理，交互流畅，视觉层次清晰，版面规整。
可靠性 在系统的使用过程中，应避免以下情况的发生：系统频繁崩溃、大量数据丢失或错乱、界面长时间的混乱或乱码、严重的安全性问题。
可扩展性 为了支持未来的业务需求和系统的可扩展性，系统应提供二次开发的能力，使得新增模块的增加不会对原有系统的结构和代码造成过多的修改。
可维护性 为了确保系统在发生崩溃或数据丢失情况下能够快速恢复正常运行，系统应该支持定期数据备份和及时的数据恢复功能。

系统总体设计

系统体系结构设计

在线生词默写系统的体系结构采用B/S体系结构。B/S体系结构是一种常见的软件体系结构模式，该体系结构已经被广泛应用于Web应用程序的开发和部署中。B/S体系结构将整个应用程序划分为两个主要组件：浏览器（Browser）和服务器（Server）。

在 B/S 体系结构中，浏览器作为客户端，提供用户界面和与用户的交互功能。用户通过浏览器访问Web应用程序，使用HTML、CSS和JavaScript等前端技术与服务器进行通信。浏览器负责将用户的请求发送到服务器，并将服务器返回的响应在用户界面上进行展示。

服务器是 B/S 体系结构的核心组件，负责处理用户请求、执行业务逻辑、访问数据库和生成动态内容。服务器接收来自浏览器的请求，处理请求并生成相应的数据或页面，然后将结果返回给浏览器。

从系统架构上看，项目被分为浏览器层、Web容器层和数据库服务器层。

浏览器层是架构中与系统有交互行为的一层，用户通过客户端浏览器来访问和操作该应用程序；系统借助 Vue.js 框架、Naive UI等设计界面显示和与用户的互动；

Web容器层为整个结构中的核心，将 Spring Boot 框架内置的 Tomcat 服务器集成在 Web 容器中，为浏览器提供计算支持；系统借助三层结构：用户接口层（Endpoint）、业务逻辑处理层（Business Logic）以及数据访问层（Data Access）设计和部署业务功能。

数据库服务器层位于系统结构的最底层，被认为是系统中至关重要的一层。在现代信息时代，数据被视为无价的资产，因此数据库服务器层的功能显得尤为重要。该层主要集成了与数据库相关的服务，为Web应用程序提供了数据存储、查询和管理等核心功能。数据库服务器层在系统结构中扮演着关键的角色。它集成了数据库相关的服务，为Web应用程序提供了可靠的数据存储、高效的查询功能和安全的数据管理。通过数据库服务器层，数据可以得到有效管理、保护和利用，为用户和应用程序提供了强大的数据支持。
